## Transcript 1: Gemini 3.1 One-Shot Cinematic Websites

### 🧠 Key Takeaways for $ANDY:
1.  **The "Cinematic" Prompt Structure:** To get top-tier results from the AI generator, we must use specific role-playing prompts. 
    *   *Example keywords:* "world-class senior creative technologist," "architect a high-fidelity cinematic 1:1 pixel-perfect landing page."
2.  **Aesthetic Contrast:** The video highlights "high-end organic tech clinical boutique" or "brutalist luxury." Applying a hyper-premium, serious luxury design to a dog's coin creates immediate comedic value.
3.  **Micro-Animations:** The difference between a cheap site and a premium site is in the details: scroll animations, background blurs that trigger on zoom/scroll, and hover states that "glow." We must include these.
4.  **Commoditization of Design:** The AI can generate the entire layout, so our focus should be on the *content* (Andy's personality, the "Talk to Andy" AI chat) rather than struggling with CSS grids manually.

## Transcript 2: Combining Google Stitch and Anti-Gravity

### 🧠 Key Takeaways for $ANDY:
1.  **AI-Driven UX/UI Iteration:** "Google Stitch" is highlighted as an incredible UI/UX model for doing one-shot multi-page generation. The workflow here involves taking a design and then refining it page-by-page. While we are building a single-page site for $ANDY, this means we can generate the base layout, and then have the AI specifically enhance sections (like making the "checkout" or in our case "Buy $ANDY" button glow/animate).
2.  **Hosting & Deployment:** The video specifically calls out using **Vercel** via a GitHub repository. Vercel is free, incredibly fast, and updates automatically whenever code is pushed to GitHub. This validates our plan: we will code the HTML/CSS as a static app, push it to GitHub, and link it to Vercel for free, instantly updating hosting.
3.  **No Pure Templates:** The tutorial emphasizes that you don't just use a rigid template. You give it brand guidelines (for us: "brutalist luxury," grungre, dog meme) and the AI designs around that identity.
4.  **Local to Cloud Workflow:** Develop on local host -> Review -> Push to GitHub -> Vercel auto-deploys. This is the exact workflow we will use to deploy Andy's site.
