# $ANDY Website: Brain & Execution Plan

## 1. Global Context ("The Brain")
This is the core identity of the website. Every component, line of copy, and animation must align with this.

*   **Persona:** Andy is a grumpy, anti-social, cynical Husky on the Solana blockchain. He doesn't want you to buy his coin, and he definitely doesn't want to talk to you.
*   **Design Aesthetic:** "Brutalist Luxury" / "High-end Cinematic." The site shouldn't look like a cheap meme coin. It should look like an ultra-premium tech product (think Apple or a high-end AI lab), making the contrast of a grumpy dog incredibly funny.
*   **Color Palette:** Monochrome/Dark Mode with harsh, clinical contrasts. Deep blacks, crisp whites, and maybe one stark accent color (like a neon cyber-yellow or aggressive red) for buttons.
*   **Typography:** Sleek, modern sans-serif (e.g., Inter, Roboto, or Space Grotesk) to emphasize the "clinical boutique" feel.

## 2. Technical Architecture
*   **Tech Stack:** Vanilla HTML, CSS, JS. No heavy frameworks needed for a single-page landing site.
*   **Structure:**
    *   `index.html`: The main structure.
    *   `styles.css`: All styling, animations, and brutalist luxury rules.
    *   `script.js`: Logic for interactive elements (Chat widget, animations).
*   **Deployment:** Local testing -> GitHub Repo -> Vercel (Auto-deploy).

## 3. Core Features & Layout (Single Page)
*   **Hero Section:** High-impact. A cinematic image of Andy (looking unamused). Large brutalist typography stating something dismissive ("Leave me alone." or "Don't buy $ANDY.").
*   **Interactive Chat Widget:** The centerpiece. A chat interface where users try to talk to the AI Andy, and he responds with grumpy dismissals.
*   **"Grump-o-Meter":** A dynamic/interactive element. As the user scrolls or hovers over things, a meter fills up showing how annoyed Andy is getting with their presence.
*   **Tokenomics/Info Section:** Presented clinical and sterile. 
*   **Footer/Links:** DexScreener, Twitter, Telegram links.

## 4. Execution Plan (Step-by-Step)
1.  **Phase 1: Foundation.** Create the initial file structure (`index.html`, `styles.css`, `script.js`). Set up the global CSS variables (colors, fonts).
2.  **Phase 2: Hero & Aesthetic.** Build the high-fidelity hero section with premium scroll reveals and frosted glass effects.
3.  **Phase 3: The Chat Widget.** Build the UI for the interactive chat and the javascript logic to make it functional (triggering grumpy responses).
4.  **Phase 4: Optimization.** Apply the UI/UX checklist (custom cursors, lazy loading, micro-animations on hover).
5.  **Phase 5: Deployment.** Push code to GitHub and deploy to Vercel.
